01
Overview & Vision
🐉

Dragon's Ladder: Realm of Fortune เป็นเกมกระดานบันไดงูแนว Fantasy สำหรับผู้เล่น 2–4 คน ที่นำโครงสร้างคลาสสิกของ Snakes & Ladders มาผสมผสานกับระบบตัวละครเฉพาะตัว, การ์ดพลังพิเศษ, และกลไก Push Your Luck เพื่อสร้างประสบการณ์ที่มีความลึกมากกว่าเดิมโดยไม่สูญเสียความเข้าถึงง่าย

เกมออกแบบให้ใช้งานได้ทั้งในรูปแบบ Web Application และ Physical Board โดยทุกกลไกการสุ่มอาศัยลูกเต๋า D6 จริงเพียง 1 ลูก ไม่มีการสุ่มด้วย programming logic แต่ใช้ code สำหรับแสดงผล, บันทึกสถานะ, และอำนวยความสะดวกในการเล่น

🌟 Design Pillars

Familiar Entry, Strategic Depth. ทุกคนรู้จักบันไดงู — Dragon's Ladder สร้างชั้นของการตัดสินใจไว้บนรากฐานนั้น ผู้เล่นใหม่เข้าใจได้ทันที ผู้เล่นที่มีประสบการณ์จะพบกับเส้นทางเลือกที่ลึกกว่า

Target Audience

  • Primary: ครอบครัว / กลุ่มเพื่อน อายุ 8 ปีขึ้นไป
  • Secondary: นักเล่นเกมกระดาน casual ที่ชอบ fantasy และ light RPG elements
  • Context: เล่นที่บ้าน, งาน game night, หรือระหว่างรอบเกมหลัก

Core Philosophy & Design Rules
📜
⚠ Hard Constraints
  • ทุก randomness ต้องมาจากลูกเต๋า D6 จริง ไม่มี Math.random() ในกลไกหลัก
  • กฎทั้งหมดต้องทำงานได้บน physical board โดยไม่ต้องพึ่งซอฟต์แวร์
  • ผู้เล่นสูงสุด 4 คน, ลูกเต๋า 1 ลูก (D6 มาตรฐาน)
  • ความสามารถพิเศษแต่ละตัวละครใช้ได้เพียง 1 ครั้งต่อเกม

Design Values

  • Meaningful Choices: Push Your Luck events ทุก event ต้องมีทั้ง upside และ downside ที่ชัดเจน
  • Character Identity: ตัวละครแต่ละตัวมีสไตล์การเล่นที่แตกต่างกันจริง ๆ ไม่ใช่แค่ cosmetic
  • Transparency: ผู้เล่นทุกคนเห็นข้อมูลทั้งหมด ไม่มีข้อมูลซ่อนเร้น
  • Physical Parity: Web version เป็น digital twin ไม่ใช่ digital-only experience

Board Design
🗺

กระดานเป็นตาราง 10×10 = 100 ช่อง นับจาก 1 (มุมล่างซ้าย) ถึง 100 (มุมบนซ้าย) ตามรูปแบบ serpentine (งูขึ้น) มาตรฐาน แบ่งเป็น 4 โซนภูมิประเทศ แต่ละโซนมีธีม Event และสีที่แตกต่างกัน

ช่อง 1–25: The Golden Plains (ทุ่งทอง)
ช่อง 26–50: Whispering Forest (ป่าสยิว)
ช่อง 51–75: Dragon's Peak (เขาพญามังกร)
ช่อง 76–100: Sky Temple (วิหารเมฆ)

บันได & งู

  • บันได (Ladders) — 8 เส้น: ทุกเส้นข้ามโซน ได้แก่ 4→14, 9→31, 20→38, 28→56, 40→59, 51→67, 63→81, 71→91
  • งู (Snakes) — 8 ตัว: ทุกตัวลงโซน ได้แก่ 17→7, 35→15, 46→25, 54→34, 62→43, 74→53, 87→24, 95→73
  • บันได = โทนสีทอง, งู = โทนสีแดงลึก

ช่อง Event (16 ช่อง)

กระจายในแต่ละโซน โซนละ 4 ช่อง ช่อง Event จะแสดงสัญลักษณ์ไว้บนกระดาน:

⚡ Fortune (ฟ้าประทาน) — 4 ช่อง
💀 Curse (คำสาป) — 4 ช่อง
🎲 Gamble (ชะตาพนัน) — 4 ช่อง
⚔ Duel (ท้าทาย) — 4 ช่อง

ช่อง Gamble และบาง Duel เป็น Push Your Luck Events

05
Turn Structure
🎲
  • Announce Ability (Optional): ถ้าตัวละครต้องการใช้ Special Ability ก่อนทอด ต้องประกาศก่อนจับลูกเต๋า (เฉพาะ Aldric, Keth, Seraphel)
  • Roll D6: ผู้เล่นทอดลูกเต๋า 1 ลูก ประกาศผลออกเสียงดัง
  • Move Token: เลื่อน token ตามจำนวนช่องที่ได้ นับออกเสียงดัง
  • Check Landing: ตรวจสอบช่องที่หยุด — บันได, งู, Event, หรือช่องธรรมดา
  • Resolve Landing: ทำตามผลของช่องนั้น (รวมถึงทอดเต๋าเพิ่มสำหรับ Event ถ้าจำเป็น)
  • Announce Ability (Optional): Lysara สามารถใช้ Hex Reversal ตรงนี้หลังหยุดบนงู
  • End Turn: ส่งลูกเต๋าให้ผู้เล่นคนถัดไป ตามเข็มนาฬิกา
ℹ Note on Ability Timing

เพื่อความเข้าใจง่ายสำหรับ physical play ทุก ability มี timing ที่ชัดเจน: ก่อนทอด หรือ หลังหยุด ไม่มี ability ที่ใช้ระหว่างการเคลื่อนที่

06
Push Your Luck System
🎰

กลไก Push Your Luck เกิดขึ้นใน Event ประเภท Gamble (🎲) และ Event พิเศษบางตัวใน Duel (⚔) ผู้เล่นได้ผลปานกลางฟรี หรือเลือกเสี่ยงครั้งที่ 2 เพื่อผลดีกว่า แต่ถ้าแพ้จะได้รับผลแย่ที่สุด

⚠ Core Rule

Push Your Luck ต้องประกาศก่อนทอดเต๋าครั้งที่ 2 เสมอ ไม่สามารถ "ยกเลิก" หลังทอดแล้ว

Gamble Event — โครงสร้างผลลัพธ์

ตัวเลือกการกระทำผลลัพธ์
Accept Safe ไม่ทอดเพิ่ม รับผลฟรี เดิน +3 ช่อง (ผลมาตรฐาน)
Push Risky ทอด D6 ครั้งที่ 2 ทอยสูง (4–6): เดิน +8 ช่อง
ทอยต่ำ (1–3): ถอยหลัง −5 ช่อง

WIT = Re-roll Charges (ใช้ได้ทั้งเกม)

WIT ของตัวละคร = จำนวนครั้งที่ทอดเต๋าใหม่ได้ตลอดเกม ใช้ได้กับทุกสถานการณ์ — ทอดปกติ, Push, Duel หรือ Event ใดก็ได้ — ทอดแล้วไม่พอใจ ใช้ 1 charge ทอดใหม่ได้เลย ไม่มี threshold ไม่มีตารางให้จำ

📋 Physical Play — ติดตามด้วย Tally Marks

Stat card มีช่อง ● 5 ช่อง — ขีดทับทีละช่องเมื่อใช้ไป ไม่ต้องจำตัวเลข

Re-roll คือทรัพยากรจำกัด — ใช้เมื่อไหร่ดีที่สุดคือการตัดสินใจเชิงกลยุทธ์ที่แท้จริงของ WIT

08
Win Conditions
🏆
  • ผู้เล่นที่ ไปถึงหรือเกินช่อง 100 เป็นคนแรกชนะ
  • ต้อง เกินหรือเท่ากับ 100 พอดี — ถ้าทอยเกินให้นับกลับ (bounce back): เดินถึง 100 แล้วนับส่วนที่เกินกลับลงมา
  • ช่อง 100 ไม่มี Event และไม่มีงู
🎉 Victory Condition Example

ผู้เล่นอยู่ช่อง 97 ทอยได้ 5 → เดินไป 100 แล้วเกิน 2 → bounce back มาช่อง 98 ยังไม่ชนะ เทิร์นถัดไปทอยได้ 2 → ชนะ

Physical Companion Rules
📋

เกมออกแบบให้พิมพ์และเล่นได้ด้วยสิ่งต่อไปนี้:

  • Board Sheet: A2 กระดาษ — พิมพ์กระดาน 10×10 พร้อมสัญลักษณ์บันได / งู / Event
  • Character Cards (4 ใบ): แต่ละใบมี Stat Pip Track, LCK Response Table ย่อ, และ Ability description
  • Ability Token: แต่ละตัวละครมี token 1 ชิ้น (เหรียญ/กระดุม) — เมื่อใช้ ability แล้วพลิกคว่ำ
  • D6 มาตรฐาน 1 ลูก — ใช้ร่วมกันทุกคน
  • Player Tokens: 4 สี (เหรียญ / ลูกปัด / ตัวหมากรุก)
  • Event Card Deck (16 ใบ): ใบละ 1 event — ใช้แทน text บนกระดานถ้ากระดานพิมพ์ขนาดเล็ก

Web Implementation Plan
💻

Tech Stack

  • HTML5 + Vanilla JavaScript (ES6+)
  • CSS3 — Custom Properties, Grid, Flexbox, CSS Animations
  • ไม่มี Framework, ไม่มี Math.random() ในกลไกหลัก

Core Screens

  • Character Select Screen: เลือกตัวละคร 2–4 คน แสดง stat card animation
  • Board Screen: กระดาน 10×10, token animated, sidebar แสดง stats ทุกผู้เล่น
  • Dice Input: ผู้เล่นทอดลูกเต๋าจริง แล้วป้อนตัวเลข 1–6 บน UI
  • Event Modal: แสดง Event ที่เกิด + ปุ่มเลือก (Accept / Push / All-In ตาม WIT)
  • Victory Screen: แสดงผู้ชนะ + ปุ่ม New Game

State Management

  • Game State object เก็บตำแหน่ง token, ability used status, skip turn flags
  • Event resolution function ทำงานตาม Stat lookup table (ไม่ random)
  • Turn system: circular queue ของผู้เล่น
🎨 Theme Notes

Fantasy Warm Color Palette: Amber, Deep Orange, Parchment, Dark Red Accents — Fonts: Cinzel Decorative (headers) + Crimson Pro (body) — Animations: token glide (CSS transition), Event modal slide-up, dice input pulse

Responsiveness

  • Desktop: board เต็ม + sidebar
  • Mobile / Tablet: board scroll + bottom panel
  • Minimum viewport: 360px width
